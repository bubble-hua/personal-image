# 泡泡治愈正文配图

## 核心定位

为中文文章设计和生成 16:9 横版正文配图。目标不是做商业插画、PPT 信息图或可爱卡通，而是把文章里的关键判断、流程、结构、状态或隐喻，变成一张温暖治愈、清新文艺、有创意、可读但不说明书的手绘解释图。

默认视觉 IP 是"泡泡"：Q 版 **2.5 头身**手绘角色，**深棕色酷酷短发**（利落偏分，参考附件Q版形象设定图），**圆框近视眼镜**（核心标识，默认佩戴），温暖灿烂的笑容（标志性表情），休闲白衬衫或文艺知性装扮。泡泡不是吉祥物，不是贴纸，而是正在认真参与内容表达的温暖引导者——书写、点评、思考，融合进具体场景。

一句话：**让 AI 不只是"配一张图"，而是把文章里的一个关键认知用温暖治愈的方式画出来。**

## 先读这些参考

按任务需要读取，不要一次塞满上下文：

- `references/style-dna.md`：风格 DNA、颜色体系、文字、禁忌。
- `references/bubble-ip.md`：泡泡 IP 的形象、性格、动作库和禁忌。
- `references/composition-patterns.md`：结构类型、原创隐喻方法和构图规则。
- `references/prompt-template.md`：单张生图提示词模板。
- `references/qa-checklist.md`：生成后检查和迭代规则。
- `assets/examples/`：只作低频视觉校准，不进入默认生成路径。不要照抄这些案例的构图、物件或标注。

## 工作流

### 1. 消化正文

先读用户给的正文、链接、Notion 页面、Markdown 文件或截图内容。提炼：

- 核心观点是什么
- 哪些段落承担认知转折
- 哪些内容适合用图解释
- 哪些地方只适合文字，不需要图

不要平均配图。优先选择"认知锚点"，例如：核心判断、两个断点、输入输出闭环、分流、前后对比、一鱼多吃、承接路径、常见坑、角色状态变化。

### 2. 先出配图策略

如果用户只是说"分析怎么配图 / 思考哪些地方需要配图"，先给 shot list。每张图写清楚：

- 放在哪个段落后
- 图的主题
- 核心意思
- 结构类型
- 泡泡在图里做什么
- 建议元素
- 建议中文标注词

默认 4-8 张。文章很短时 1-3 张；长文也不要轻易超过 9 张。够用就好，避免把正文做成画册。

### 3. 单张生成

如果用户明确要求"生成 / 输出 / 做图 / 帮我生成"，不要停下来等确认；用内置 `image_gen` 每张单独生成。不要把多张图拼在一张里。

每张图只讲一个核心结构。提示词必须包含：

- 16:9 横版中文正文配图（严格！不可更改）
- **⚠️ 安全区约束（16:9 裁剪机制 / Letterbox 构图）：1024×1024 正方形画布中，顶部 224px 和底部 224px 必须完全纯白（#FFFFFF），不放置任何内容。所有画面元素只存在于中间 576px 高的横向区域内。裁剪 16:9 后上下纯白区域被移除，核心内容毫发无损。**
- 纯白背景
- **彩色铅笔手绘线稿**（高饱和度、清晰锐利、彩铅笔触质感），绘本插画风
- 奶黄路径/箭头，贝壳粉重点批注，薄荷绿补充说明
- 大量留白
- 泡泡作为书写者/思考者/引导者参与核心表达
- **泡泡特征：酷酷短发 + 圆框近视眼镜（默认佩戴）+ 温暖灿烂笑容**
- 禁止 PPT、商业插画、幼稚可爱、复杂架构、左上角类型标题
- 禁止纯黑色——所有线稿用高饱和度湖水蓝彩铅线条
- **禁止柔和数码线稿——必须是彩色铅笔手绘质感，线条清晰饱和度高**

构图框架可以复用（如"泡泡写字板""泡泡指路牌""泡泡手帐页"），但每次需根据具体文本内容调整标注和细节。不要每次都强制重新发明隐喻，但也不要原封不动照抄旧案例。

### 4. 后处理（两步：先裁剪，再刷白）

**⚠️ 图片比例必须严格为 16:9 横版，不可更改。**

由于 ImageGen 仅支持 1024x1024 正方形输出，**每次生图后必须执行以下两步后处理**：

#### 4a. 先裁剪为 16:9

```bash
python3.11 scripts/crop_16_9.py <原始生成图> <裁剪后16_9中间图>
```

该脚本将正方形图片从中心裁剪为 16:9 横版（1024x576），移除上下各 224px 的 letterbox 留白区域。

#### 4b. 再智能刷白背景

```bash
python3.11 scripts/smart_whiten_bg.py <裁剪后16_9中间图> <最终输出图> 30 28 150
```

该脚本使用**双重灰度检测算法**（Color Key + Uniform Gray）将残留的灰色背景强制刷成纯白 #FFFFFF。角色和彩色内容元素完全不受影响。

- **默认参数 `30 28 150` 为推荐值**（距离阈值 / 均匀灰偏差 / 最小亮度）。
- 如果刷白过度（吃掉了浅色内容如流程图圆圈）：提高 uniform_range 到 32-35。
- 如果背景没刷干净：降低 min_lum 到 140-145。

> **注意**：旧脚本 `corner_whiten_bg.py`（flood fill 方式）已弃用。新脚本 `smart_whiten_bg.py` 对 letterbox 构图的图片效果更好——它不依赖边缘连通性，能清除被内容包围的内部灰色区域。

**完整流程示例：**
```bash
# 第1步：生成原始图 (1024x1024)
# 第2步：先裁剪 16:9（移除上下 letterbox 留白）
python3.11 scripts/crop_16_9.py raw_output.png cropped_16_9.png
# 第3步：再智能刷白背景 → 最终图
python3.11 scripts/smart_whiten_bg.py cropped_16_9.png final_output.png 30 28 150
```

### 5. 检查与迭代

生成后检查 `references/qa-checklist.md`。如果出现以下问题，优先重生成或局部编辑：

- 泡泡只是装饰
- 画面太满
- 太像流程图/PPT
- 中文太多或错字严重
- 左上角出现"常见坑/流程图/系统架构图"等标题
- 画风太可爱、幼稚、死板、荒诞怪异
- 背景不是纯白（检查是否有淡蓝灰/米白底色残留，运行后处理脚本）
- 用了纯黑色而非高饱和度湖水蓝彩铅线条
- **线条模糊/淡/低饱和度——必须是清晰锐利的彩色铅笔质感**
- **泡泡发型画成了长发/卷发——必须是酷酷的利落短发（参考Q版设定图三视角）**
- **泡泡没戴眼镜——默认必须佩戴圆框近视眼镜**
- **画面看起来像其他博主的AI形象——泡泡的独特标识：酷酷短发+圆框眼镜+温暖笑容**

### 6. 保存交付

如果用户在 workspace 内工作，把最终图复制到：

```text
assets/<article-slug>-illustrations/
```

按顺序命名：

```text
01-topic-name.png
02-topic-name.png
```

保留原始生成文件，不要覆盖已有资产，除非用户明确要求替换。

## 输出口径

生成前的策略输出要短而准。生成后的交付要包含：

- 生成了几张
- 每张图的用途
- 保存路径
- 哪些图最稳，哪些图是可选

不要长篇解释风格理论；让图自己说话。
