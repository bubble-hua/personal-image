# QA Checklist

## 必过项

- 是 16:9 横版。
- 背景是干净白底。
- 有泡泡。
- 泡泡承担核心表达角色（书写/思考/引导），不只是装饰。
- 画面温暖治愈、清新文艺、有创意。
- 简洁清爽，主体不超过画面约 60%。
- 一张图只讲一个核心结构。
- 中文标注少、短、能读。
- 线稿主体为湖水蓝色，非纯黑。
- 奶黄色只用于主路径或箭头。
- 贝壳粉只用于重点、问题、提醒或结果。
- 薄荷绿只用于补充说明、反馈或次要信息。
- 构图框架可与已有模板相似但内容和标注已根据文本调整。

## 失败信号

出现以下情况，重生成或局部编辑：

- 左上角有"常见坑 / Workflow / 系统架构图 / 路线图"等标题。
- 泡泡像吉祥物、表情包或可爱卡通。
- 泡泡只是站在角落里，没有参与内容表达。
- 画面像 PPT、课程课件、正式流程图。
- 元素太多、箭头太多、节点太多。
- 文字变成大段解释。
- 背景有纸纹、阴影、渐变、米色、噪点。
- 真实 UI 截图或科技感界面。
- 中文错字严重或标注不可读。
- 画面太死板、太商业、太精致。
- 用了纯黑色而不是湖水蓝色。
- 画面荒诞怪异而不是温暖治愈。
- 治愈元素堆砌过多，像儿童插画。

## 迭代方法

- 太普通：让泡泡成为动作主体，加入一个温暖但有趣的隐喻。
- 太复杂：删节点，只保留一个动作和 3-5 个短标注。
- 太可爱：强调 warm but not childish, artsy but not pretentious。
- 太 PPT：去掉标题、边框、整齐网格和过多箭头，改成手绘场景。
- 太像旧案例：保留核心意思，换掉主物件和泡泡动作。
- 太荒诞：换掉怪诞元素，改用治愈系场景（花园/书架/云朵/手帐）。
- 文字错：优先局部编辑；错得多就重生成并减少标注数量。

## 交付判断

高质量图应该让读者先觉得"好温暖、好清新"，然后 1 秒内看懂结构。

如果第一眼像教程页或商业插画，而不是白纸上的温暖文艺速写，就不合格。
